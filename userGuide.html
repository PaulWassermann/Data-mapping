Contexte

Il s'agissait de développer un outil de visualisation cartographique de données extraites d'un fichier .csv fourni par l'utilisateur à l'aide de mapbox GL JS.

Mode d'emploi

Pour utiliser ma solution à l’exercice proposé, il faut disposer d’un fichier .csv respectant ces deux critères :

- Le fichier doit contenir une ligne d’en-tête indiquant la nature des données de chaque colonne. Les noms de colonne figurant dans l’en-tête peuvent cependant être librement choisis par l’utilisateur

- Sur chaque ligne, les données doivent impérativement se succéder dans l’ordre suivant : nom du lieu, longitude, latitude, indicateur 1, indicateur 2, indicateur 3


Un bouton permet de charger son fichier de données sur la page internet, à la suite de quoi l’utilisateur gagne l’accès à de nouvelles fonctionnalités :

- Sur la gauche de la carte, une liste des lignes du fichier présentant une non-conformité et ne pouvant donc pas être traitées intégralement (absence d’une ou plusieurs valeurs, localisation inconnue, …). Certaines de ces données sont cliquables et permettent de naviguer, sur la carte, jusqu’au marqueur leur correspondant.

- Sur la droite, un menu permettant d’effectuer diverses opérations affectant la visualisation des données sur la carte (filtrage par indicateurs, carte thermique, …). Cliquer sur le bouton « Rafraîchir la carte » afin d’observer les modifications.


Les marqueurs présents sur la carte sont interactifs : l’utilisateur peut cliquer sur chacun d’entre eux afin de visualiser les données attachées à un couple de coordonnées géographiques.

Enfin, après la récupération de toutes les données (noter que cela peut prendre un peu de temps), l’utilisateur pourra consulter une table d’agrégation de données par départements (ou autre unité territoriale si pays étranger). Chaque ligne de cette table est cliquable et permet de naviguer vers le département correspondant.
